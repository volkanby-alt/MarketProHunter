# Daily Workflow

Use this workflow after the first preview build is downloaded and extracted.

## 1. Start controlled

Begin with a small scan before running large category scans.

```text
Pages: 1
Parallel: 1
```

After this works, move to:

```text
Pages: 3
Parallel: 3
```

## 2. Review exports

Open the newest Excel file in the output folder.

Start with:

```text
Daily Winners
```

Then check:

```text
Summary
All Products
```

## 3. Pick products

Prioritize products with:

- high Upload Score,
- positive Net Profit,
- strong image quality,
- strong title quality,
- good bullet, description, and specification scores,
- low warning notes.

## 4. Keep the target realistic

The daily target is 200 upload-ready products.

If the scan produces fewer than 200, increase category coverage or page count after confirming the app remains stable.

## 5. Save feedback

Track real-world issues for the next version:

- bad titles,
- weak images,
- wrong brand parsing,
- missing product details,
- products that should have been rejected,
- products that should have been accepted.

These notes should drive v1.1 changes.
